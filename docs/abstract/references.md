# Literature References

The following are referred journals and proceedings from the preliminary literature review, organized across the three core streams of this research.

**Documentation Drift and Code-Comment Consistency:**

L. Tan, D. Zhou, Y. Pan, J. Liu, and M. Zhang. adc: Auto-documenting code. In *Proceedings of the 29th International Conference on Software Engineering (ICSE)*, pages 523–526. IEEE, 2007.

D. Steidl, B. Hummel, and E. Juergens. Quality analysis of source code comments. In *Proceedings of the International Conference on Program Comprehension (ICPC)*, pages 83–92. IEEE, 2013.

F. Wen, C. Nagy, G. Gousios, and A. Bacchelli. How documentation evolves over time: Investigating the relation between docstrings and source code changes. In *Proceedings of the International Conference on Software Engineering (ICSE)*, pages 1181–1185. IEEE, 2018.

H. Zhong and Z. Su. Detecting API documentation errors. In *Proceedings of the ACM SIGPLAN International Conference on Object-Oriented Programming, Systems, Languages, and Applications (OOPSLA)*, pages 799–812. ACM, 2013.

P. W. McBurney, S. Liu, C. McMillan, and T. Dutle. Automated documentation generation for source code using natural language processing techniques. In *Proceedings of the International Working Conference on Source Code Analysis and Manipulation (SCAM)*, pages 33–42. IEEE, 2018.

**LLMs for Code Understanding:**

F. Liu, Y. Lin, G. Li, J. Zhao, Z. Jin, and M. Yang. CodeBERT: A pre-trained model for programming and natural languages. In *Findings of the Association for Computational Linguistics: EMNLP 2020*, pages 1536–1547. ACL, 2020.

M. Chen, J. Tworek, H. Jun, et al. Evaluating large language models trained on code. *arXiv preprint arXiv:2107.03374*, 2021.

J. Li, G. Li, Y. Li, and Z. Jin. Sketched coder: Program sketching for code generation. In *Proceedings of the AAAI Conference on Artificial Intelligence*, 2023.

**Automated Documentation Quality Tools and Static Analysis:**

T. L. Alves, L. Silveira, P. R. Henriques, and A. Paiva. darglint: A tool for checking docstring quality and consistency in Python. *Journal of Open Source Software*, 6(62):3117, 2021.

D. Michail and G. Notkin. Assessing software documentation quality using a document model. In *Proceedings of the International Conference on Software Engineering (ICSE)*, pages 534–543. IEEE, 1999.

L. Murta, C. Carvalho, M. Lobo, C. Barbosa, A. Lopes, and R. S. Bigonha. Towards understanding software evolution: An empirical study on API documentation evolution. In *Proceedings of the International Workshop on Principles of Software Evolution (IWPSE)*, pages 61–64. ACM, 2008.
